# Самонаводящиеся ракеты — Лог решений

## Обзор фичи
Добавление самонаводящихся ракет в игру Asteroids. Игрок запускает ракету клавишей R, она летит по дуге к ближайшей цели (астероид/UFO). При столкновении с любой целью по пути — цель уничтожается. После запуска включается таймер респавна ракет.

---

## Решение 1: Архитектура ECS-компонентов

**Выбор:** Создать отдельный `MissileData` компонент на корабле (по аналогии с `GunData`/`LaserData`) и отдельный `HomingData` компонент на сущности ракеты.

**Почему:**
- `MissileData` на корабле хранит состояние оружия (амуниция, перезарядка, флаг стрельбы) — паттерн идентичен `GunData`/`LaserData`
- `HomingData` на ракете хранит параметры наведения (скорость поворота) — отделяет логику наведения от движения
- `MissileTag` маркирует сущности ракет для системы коллизий

**Альтернатива:** Использовать `GunData` с модифицированным поведением — отвергнуто, т.к. ракета имеет принципиально другую механику (наведение vs прямолинейное движение).

**Использованные скилы:** Анализ существующих ECS-компонентов через чтение кода (`GunData.cs`, `LaserData.cs`, `MoveData.cs`).

---

## Решение 2: Система наведения (EcsHomingSystem)

**Выбор:** ISystem без BurstCompile, который каждый кадр ищет ближайшего врага и корректирует направление ракеты.

**Почему:**
- Нужен доступ к позициям всех врагов одновременно — это запрос по нескольким тегам (`AsteroidTag`, `UfoBigTag`, `UfoTag`)
- Количество ракет мало (1-2), поэтому производительность не критична
- ISystem (а не SystemBase) — для консистентности с другими ECS-системами игровой логики
- Без Burst — потому что нужно итерировать по нескольким запросам внутри одного цикла

**Алгоритм наведения:**
1. Для каждой ракеты с `HomingData` + `MoveData`:
2. Найти ближайшего врага (минимальное расстояние)
3. Вычислить желаемое направление к цели
4. Плавно повернуть текущее направление к желаемому (lerp/slerp с TurnSpeed * deltaTime)
5. Если целей нет — лететь прямо

**Альтернатива:** Хранить target entity в HomingData — отвергнуто, т.к. цель может быть уничтожена, и нужен fallback на ближайшую. Проще пересчитывать каждый кадр.

## Решение 3: Коллизии ракеты

**Выбор:** Добавить `PlayerMissileTag` и обрабатывать в `EcsCollisionHandlerSystem` аналогично `PlayerBulletTag`.

**Почему:**
- Ракета ведёт себя как пуля игрока в плане коллизий: уничтожает врагов, даёт очки
- Отдельный тег позволяет различать ракету от пули (для будущих расширений)
- Единая точка обработки коллизий — меньше дублирования

---

## Решение 4: Визуал ракеты

**Выбор:** Уменьшенный спрайт корабля (scale 0.5) + TrailRenderer для инверсионного следа.

**Почему:**
- Требование задачи: "уменьшенный спрайт корабля"
- TrailRenderer проще в настройке, чем ParticleSystem, и даёт хороший визуальный эффект следа
- Префаб создаётся через Unity MCP для интеграции с Unity Editor

**Обновление:** По требованию "инверсионный след из спрайта частиц" — используем ParticleSystem с текстурой частицы.

---

## Решение 5: Перезарядка ракет

**Выбор:** Модель перезарядки как у лазера — по одной ракете за период.

**Почему:**
- Лазер восстанавливает `CurrentShoots += 1` за каждый `UpdateDurationSec`
- Пушка восстанавливает сразу все `CurrentShoots = MaxShoots`
- Для ракет логичнее инкрементальная перезарядка (как лазер) — каждая ракета восстанавливается отдельно

---

## Решение 6: Input Action

**Выбор:** Добавить "Rocket" action типа Button, привязанный к клавише R.

**Почему:** Паттерн идентичен "Laser" (одноразовое нажатие → один запуск). Input System поддерживает горячее добавление биндингов.

**Использованные скилы:** Анализ `player_actions.inputactions` и `PlayerInput.cs`.

---

## Решение 7: HUD

**Выбор:** Добавить два новых ReactiveValue в HudData: `MissileShootCount` и `MissileReloadTime` + `IsMissileReloadTimeVisible`.

**Почему:** Точное повторение паттерна лазера (`LaserShootCount`, `LaserReloadTime`, `IsLaserReloadTimeVisible`). Обновление через `ObservableBridgeSystem`.

---

## Решение 8: TDD-подход

**Выбор:** Сначала тесты, потом реализация для каждой системы.

**Порядок:**
1. `MissileSystemTests` — перезарядка, стрельба, нет амуниции
2. `HomingSystemTests` — поворот к цели, отсутствие целей, смена цели
3. `CollisionHandlerTests` — ракета + астероид, ракета + UFO, ракета + корабль (не должна)

**Использованные скилы:** Анализ `GunSystemTests.cs`, `AsteroidsEcsTestFixture.cs` для паттернов тестирования.

---

## Решение 9: Время жизни ракеты

**Выбор:** Использовать существующий `LifeTimeData` компонент и `EcsLifeTimeSystem` + `EcsDeadByLifeTimeSystem`.

**Почему:** Система уже есть и работает для пуль. Ракета тоже должна исчезать через N секунд, если не попала в цель.

---

## Решение 10: Обработка уничтожения ракеты

**Выбор:** В `Application.OnDeadEntity` добавить обработку `EntityType.Missile` — просто эффект взрыва.

**Почему:** По аналогии с `EntityType.Bullet` — ракета при уничтожении не порождает осколки, но может показать эффект.

- Компиляция: без ошибок

### Исправленные баги при интеграции

**Баг 1: Циклическая зависимость ECS-систем (IndexOutOfRangeException)**
- **Причина:** `EcsHomingSystem` имел `[UpdateAfter(typeof(EcsMissileSystem))]` + `[UpdateBefore(typeof(EcsMoveSystem))]`, что создавало цикл: `EcsMoveSystem → EcsShipPositionUpdateSystem → EcsGunSystem → EcsLaserSystem → EcsMissileSystem → EcsHomingSystem → [UpdateBefore] EcsMoveSystem`
- **Решение:** Убран `[UpdateBefore(typeof(EcsMoveSystem))]`. Наведение применяется с задержкой в 1 кадр — для ракеты незаметно.
- **Использованные скилы:** Анализ цепочки `[UpdateAfter]`/`[UpdateBefore]` атрибутов всех ECS-систем.

**Баг 2: Missing script на Main Camera**
- **Причина:** Не связана с фичей ракеты — уже существовала в сцене.
- **Решение:** Удалён missing MonoBehaviour через `GameObjectUtility.RemoveMonoBehavioursWithMissingScript()` + сцена сохранена.
- **Использованные скилы:** Unity MCP (script-execute) для программной очистки сцены.

**Баг 3: Фиолетовые квадраты вместо частиц (ParticleSystem trail)**
- **Причина:** При создании префаба через MCP script-execute, материал ParticleSystemRenderer оказался `null`. Шейдер `Sprites/Default` не совместим с URP pipeline проекта.
- **Решение:** Создан материал `MissileTrailMat.mat` с шейдером `Universal Render Pipeline/Particles/Unlit` + текстура `Default-Particle.png`. Материал сохранён в `Assets/Media/materials/`.
- **Использованные скилы:** Unity MCP (script-execute) для диагностики шейдеров и создания URP-совместимого материала.

**Баг 4: EntityFactory.CreateShip — отсутствующие параметры**
- **Причина:** Добавление `MissileData` в `CreateShip` изменило сигнатуру, а `EntityFactoryTests` не были обновлены.
- **Решение:** Обновлены все 3 вызова `CreateShip` в тестах + добавлены тесты `CreateMissile`.
