# Режим `list` — список задач и эпиков

Цель: показать задачи и эпики из `.planning/tasks/` с фильтрами. Только чтение.

## Шаги

### 1. Парсинг аргумента

Аргумент после `list`:

| Аргумент | Вывод |
|----------|-------|
| пусто | Группированный по эпикам, все активные (не `done`) |
| `all` | Группированный, включая `done` из `archive/` |
| `epics` | Только таблица эпиков |
| `E-NN` или `NN` с буквой `E` | Задачи конкретного эпика (плоская таблица) |
| `ready`, `executing`, `review`, `revise`, `done` | Фильтр по статусу (плоская) |
| `<слово>` | Подстрочный поиск по `title`/`slug` (плоская) |

Для коротких ID (`1` → `E-01`, `2` → `E-02`) — нормализация по правилам SKILL.md. Если аргумент — просто цифра без буквы `E` и не матчит статус — трактуй как подстрочный поиск (цифра может быть частью ID задачи). Исключение: если из контекста ясно, что пользователь имел в виду эпик — спроси для уточнения.

### 2. Инициализация

Если `INDEX.md` не существует — «Индекс задач пуст. Создай первую задачу через `/task new`.» Стоп.

### 3. Сбор данных

Источник истины — frontmatter'ы TASK.md. INDEX.md — представление.

1. Прочитай таблицу `## Эпики` из INDEX.md — список с метаданными и статусами.
2. Активные задачи: `Glob` `.planning/tasks/T-*/TASK.md`. Для каждой прочитай frontmatter: `id`, `status`, `slug`, `title`, `created`, `epics`, `related`.
3. Архив (только если фильтр = `all`, `done`, или явный поиск по слову/эпику): `Glob` `.planning/tasks/archive/*/TASK.md`, аналогично.

### 4. Вывод

#### Пусто / `all` — группированный

```
Всего: 12 задач (ready: 3, executing: 1, review: 2, revise: 1, done: 5) в 3 эпиках.

### E-01 — Корабль и движение (active)
| ID     | Статус    | Заголовок        | Создана    |
|--------|-----------|------------------|------------|
| T-0001 | ready     | WASD-управление  | 2026-04-18 |
| T-0004 | done      | Инерция          | 2026-04-19 |

### E-02 — Боевая основа (active)
| ID     | Статус    | Заголовок            | Создана    |
|--------|-----------|----------------------|------------|
| T-0003 | executing | Стрельба из орудия   | 2026-04-18 |

### Без эпика
| ID     | Статус | Заголовок           | Создана    |
|--------|--------|---------------------|------------|
| T-0002 | ready  | Исследование UI лога| 2026-04-18 |
```

Задача с несколькими эпиками — дублируется в каждой подсекции.
Без `all` — исключи задачи со статусом `done`.

#### `epics` — только таблица эпиков

Для каждого эпика посчитай количества из frontmatter'ов (активных и общее с архивом).

```
| ID   | Slug          | Название          | Задач (активных/всего) | Статус |
|------|---------------|-------------------|------------------------|--------|
| E-01 | ship-movement | Корабль и движение| 3 / 5                  | active |
| E-02 | combat-core   | Боевая основа     | 0 / 4                  | closed |
```

#### `E-NN` — задачи эпика

Нормализуй E-NN. Если нет в таблице — «Эпик E-NN не найден.» Стоп.

Отбери все задачи где `E-NN ∈ epics` (активные + архив).

```
Эпик E-01 «Корабль и движение» — 5 задач:

| ID     | Статус  | Заголовок        | Создана    | Эпики         |
|--------|---------|------------------|------------|---------------|
| T-0001 | ready   | WASD-управление  | 2026-04-18 | E-01          |
| T-0005 | done    | Дрейф и инерция  | 2026-04-19 | E-01, E-02    |
```

#### Статус (`ready`/`executing`/…) — плоская

```
Статус `ready` — 3 задачи:

| ID     | Эпики       | Заголовок        | Создана    |
|--------|-------------|------------------|------------|
| T-0001 | E-01        | WASD-управление  | 2026-04-18 |
| T-0002 | —           | Исследование UI  | 2026-04-18 |
```

#### Слово — плоская с подсветкой совпадений

```
Поиск «корабль» — 2 совпадения:

| ID     | Статус | Эпики | Заголовок            | Создана    |
|--------|--------|-------|----------------------|------------|
| T-0001 | ready  | E-01  | WASD-управление      | 2026-04-18 |
| T-0005 | done   | E-01  | Инерция корабля      | 2026-04-19 |
```

Если 0 совпадений — «Нет задач по фильтру `<фильтр>`.»
