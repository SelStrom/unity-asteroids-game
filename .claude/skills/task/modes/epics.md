# Режим `epics` — эпики конкретной задачи

Формат: `/task epics <id> [операция] [E-NN,...]`

`<id>` поддерживает короткую форму (`357` → `T-0357`). Нормализация — по правилам SKILL.md.

Операции: `set`, `add`, `remove`, `clear`, или без операции (интерактив).

## Общий префлоу

1. Нормализуй `<id>` → `T-NNNN`. Если не число и не `T-NNNN` — сообщи: «Ожидаю ID задачи, получил `<arg>`.» Стоп.
2. Найди папку задачи:
   - Активная: `Glob .planning/tasks/T-NNNN-*/TASK.md`
   - Архив: `Glob .planning/tasks/archive/T-NNNN-*/TASK.md`
   - Не найдено — «Задача T-NNNN не найдена.» Стоп.
3. Прочитай TASK.md, извлеки текущие `epics: [...]` из frontmatter.

## Без операции — интерактив

Покажи:

```
T-NNNN «<заголовок>»
Текущие эпики: E-01, E-02 (или «без эпика»)

Что делаем?
- «set E-01,E-03» — заменить список
- «add E-03» — добавить
- «remove E-01» — убрать
- «clear» — снять все эпики
```

Жди ответ, обработай как соответствующую операцию.

## `set E-NN,E-NN,...`

1. Нормализуй каждый E-NN.
2. Для КАЖДОГО E-NN проверь существование в таблице `## Эпики` INDEX.md:
   - Нет — спроси: «Эпик E-NN не существует. `create` — создать inline / `skip` — пропустить этот / `cancel` — отменить всю операцию».
   - `create` — inline-создание (название, slug, описание, `E-NN` = `max + 1`), добавь строку в `## Эпики`.
   - `skip` — исключи этот E-NN из списка, продолжай с остальными.
   - `cancel` — стоп без изменений.
3. Запиши новый список в `epics:` frontmatter'а TASK.md (перепиши поле целиком).
4. Вызови утилиту «Перестроение секции Задачи» (SKILL.md).
5. Сообщи: «Эпики T-NNNN: E-01, E-02.» Или «Задача теперь без эпика.» если список пустой.

## `add E-NN`

1. Нормализуй E-NN.
2. Проверь существование в `## Эпики`. Нет — как в `set` (create/skip/cancel).
3. Если уже в `epics:` — «Эпик E-NN уже привязан к T-NNNN.» Стоп без изменений.
4. Добавь E-NN в конец списка `epics:`.
5. Ребилд секции Задачи.
6. Сообщи: «Добавлен эпик E-NN. Текущие: E-01, E-02.»

## `remove E-NN`

1. Нормализуй E-NN.
2. Если нет в `epics:` — «Эпик E-NN не был привязан к T-NNNN.» Стоп.
3. Удали E-NN из списка.
4. Ребилд.
5. Сообщи. Если список стал `[]` — дополни: «Задача теперь без эпика.»

## `clear`

1. Задай `epics: []` в frontmatter.
2. Ребилд.
3. Сообщи: «Все эпики сняты с T-NNNN. Задача в секции «Без эпика».»

## Запрещено

- Git-операции.
- Менять любые frontmatter-поля кроме `epics:`.
- Удалять строки эпиков из таблицы `## Эпики` INDEX.md при `remove`/`clear` — эпики остаются, просто задача с ними больше не связана.
- Самовольно создавать несуществующие эпики без подтверждения (`create` от пользователя).
