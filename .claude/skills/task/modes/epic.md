# Режим `epic` — управление эпиками

Субкоманды: `new`, `list`, `close`, `reopen`, `rename`.

Формат: `/task epic <субкоманда> [аргументы]`

E-NN можно указывать сокращённо (`1` → `E-01`). См. SKILL.md.

## Общая инициализация

Перед любой субкомандой — убедись что `INDEX.md` существует (создай из шаблона при необходимости).

## `/task epic` без субкоманды

Спроси: «Что делаем с эпиками — `new`, `list`, `close`, `reopen` или `rename`?». Жди.

## `/task epic new [название]`

1. Если название не передано — спроси «Название эпика?».
2. Спроси «Slug?» (предложи default: транслит названия + kebab-case).
3. Спроси «Краткое описание (1 строка)?».
4. Определи `E-NN`: прочитай таблицу `## Эпики` в INDEX.md, найди max `E-NN`, новый = max + 1, 2 цифры.
5. Добавь строку в таблицу:
   ```
   | E-NN | <slug> | <название> | <описание> | active |
   ```
6. Сообщи: «Создан эпик E-NN «<название>».»

## `/task epic list`

1. Прочитай таблицу `## Эпики` из INDEX.md.
2. Для каждого E-NN посчитай:
   - Активных задач (не `done`): `Glob` `.planning/tasks/T-*/TASK.md`, фильтр по `E-NN ∈ epics` и `status != done`.
   - Всего: то же + `archive/*/TASK.md`.
3. Перед выводом — вызови утилиту «Перестроение секции Задачи» (SKILL.md), чтобы секция была актуальной.
4. Покажи:

```
Эпиков: 3 (active: 2, closed: 1)

| ID   | Slug          | Название          | Задач (активных/всего) | Статус |
|------|---------------|-------------------|------------------------|--------|
| E-01 | ship-movement | Корабль и движение| 3 / 5                  | active |
| E-02 | combat-core   | Боевая основа     | 2 / 4                  | active |
| E-03 | meta-prep     | Подготовка меты   | 0 / 3                  | closed |
```

## `/task epic close <E-NN>`

1. Нормализуй E-NN.
2. Найди строку в таблице `## Эпики`. Нет — «Эпик E-NN не найден.» Стоп.
3. Если уже `closed` — «Эпик E-NN уже закрыт.» Стоп.
4. Собери все задачи с этим эпиком (активные + архив). Прочитай frontmatter'ы `status` и `epics`.
5. Если есть задачи в статусе не `done`:
   - Покажи: «Эпик E-NN нельзя закрыть. Незавершённые задачи:»
     ```
     - T-0001 (ready)    «WASD-управление»
     - T-0003 (executing) «Стрельба из орудия»
     ```
   - Стоп.
6. Все `done` — обнови статус в таблице: `active` → `closed`.
7. Сообщи: «Эпик E-NN закрыт.»

## `/task epic reopen <E-NN>`

1. Нормализуй E-NN.
2. Найди в таблице. Нет — стоп.
3. Если `active` — «Эпик E-NN уже active.» Стоп.
4. Обнови статус в таблице: `closed` → `active`.
5. Сообщи.

## `/task epic rename <E-NN>`

1. Нормализуй E-NN.
2. Найди строку. Нет — стоп.
3. Покажи текущие поля (название, slug, описание).
4. Спроси: «Что менять — `title` / `slug` / `description` / `all`? («skip» — отмена)».
5. Для каждого выбранного поля запроси новое значение.
6. Обнови строку в таблице `## Эпики`.
7. Если slug/title менялись — вызови утилиту «Перестроение секции Задачи» (заголовки подсекций изменились).
8. Сообщи: «Эпик E-NN обновлён.»

## Запрещено

- Git-операции.
- Удалять строки из таблицы `## Эпики`. Только `close`/`reopen`. Для удаления — ручная правка пользователем.
- Трогать frontmatter'ы задач при `close`/`reopen` эпика — ссылки `epics: [E-NN]` остаются валидными, просто эпик в статусе `closed`.
