# Режим `new` — создание новой задачи

Цель: получить описание → найти похожие → выбрать эпик → обсудить → выработать план → записать `TASK.md` со статусом `ready`.

**Git не трогать. Ни на одном шаге.**

## Шаги

### 1. Получить описание

- Если после `new` есть текст — это описание.
- Если пусто — напиши: «Опиши задачу одним сообщением.» и жди следующего сообщения.

### 2. Инициализация индекса

Проверь `.planning/tasks/INDEX.md`. Если нет — создай из `templates/INDEX.md.tmpl` (`mkdir -p .planning/tasks` при необходимости).

### 3. Поиск похожих задач

Цель: не создавать дубликат.

1. Извлеки 3–7 ключевых терминов из описания.
2. Собери кандидатов:
   - `Grep` по `.planning/tasks/INDEX.md` для каждого термина.
   - `Grep` по `.planning/tasks/**/TASK.md` (включая `archive/`) — поля `title:` и `## Цель`.
3. Отбери top-5 по совпадению.
4. Кандидатов 0 — переходи к шагу 4, ничего не спрашивая.
5. Есть кандидаты — покажи:

```
Нашёл похожие задачи:
1. T-0015 [done]     «Движение корабля WASD»
2. T-0023 [ready]    «Стрельба основного орудия»

Что делаем?
a) Расширяем существующую — напиши «a 1»
b) Новая со ссылкой — напиши «b 2»
c) Новая без ссылки — «c»
```

Жди ответ.

- `a N` → **НЕ создавай новую**. Сообщи: «Открой T-NNNN через `/task do <NNNN>` или `/task review <NNNN>`. Если нужна правка плана — скажи, расширю секцию «Доработки».» Остановись.
- `b N` → новая, в `related` → T-NNNN выбранной. Продолжай.
- `c` → новая, `related: []`. Продолжай.

### 4. Выбор эпика

1. Прочитай таблицу `## Эпики` из INDEX.md.
2. Покажи:

```
Эпики:
1. E-01 ship-movement — Корабль и движение
2. E-02 combat-core — Боевая основа

К какому эпику привязать?
- номер(а) через запятую: «1» или «1, 2»
- «new» — создать новый эпик
- «skip» — без эпика
```

Жди ответ.

3. **«new»** — inline-создание эпика:
   - Спроси: «Название эпика?».
   - Спроси: «Slug?» (предложи default от названия: транслит + kebab-case).
   - Спроси: «Краткое описание (1 строка)?».
   - Определи `E-NN` = `max(E-NN в таблице) + 1`, 2 цифры с ведущим нулём.
   - Добавь строку в таблицу `## Эпики` INDEX.md:
     ```
     | E-NN | <slug> | <название> | <описание> | active |
     ```
   - Используй созданный E-NN.
   - Спроси: «Добавить ещё эпик к этой задаче?». Если да — вернись к началу шага 4.

4. **Номера через запятую** — резолв в список E-NN. Запомни для `epics:`.
5. **«skip»** — `epics: []`.

### 5. Чтение документации

Прочитай для контекста:
- `CLAUDE.md` — правила проекта.
- `.planning/pre-production/INDEX.md` — если есть.
- Из `.planning/pre-production/` — только тематически релевантные описанию (архитектура, дизайн, API). Не всё подряд.

Зафиксируй список прочитанных файлов для секции «Изученная документация».

### 6. Формирование вопросов

Сформулируй уточняющие вопросы:
- Что неоднозначно в формулировке?
- Какие архитектурные решения требуют выбора?
- Граничные случаи?
- Зависимости на другие модули?
- Что вне scope?

Нет вопросов — скажи: «Вопросов нет, задача однозначна. Перехожу к плану.» и переходи к шагу 7.
Есть — задай списком (1 сообщение, нумерованные). Жди ответы. При необходимости follow-up.

### 7. Выработка плана

Checklist `- [ ] Шаг N — описание`. Конкретные действия, не абстракции.

Покажи план + резюме решений. «План ок? Правки?». Правки — внеси, покажи снова. Ок — переходи к 8.

### 8. Создание файлов

1. **ID задачи:** сканируй `INDEX.md` + `archive/**/TASK.md` на все `T-NNNN`, новый = max + 1, 4 цифры.
2. **Slug задачи:** 2–5 слов из заголовка, транслит (а→a, б→b, в→v, г→g, д→d, е→e, ё→yo, ж→zh, з→z, и→i, й→y, к→k, л→l, м→m, н→n, о→o, п→p, р→r, с→s, т→t, у→u, ф→f, х→h, ц→ts, ч→ch, ш→sh, щ→sch, ъ→, ы→y, ь→, э→e, ю→yu, я→ya), kebab-case, lowercase.
3. **Папка:** `mkdir -p .planning/tasks/T-NNNN-<slug>/`.
4. **TASK.md** из `templates/TASK.md.tmpl`:
   - frontmatter: `id`, `slug`, `title`, `created`, `status: ready`, `epics: [...]` (из шага 4), `related: [...]` (из шага 3 b-вариант или `[]`).
   - `## Цель` — формулировка пользователя.
   - `## Связанные задачи` — ссылки на T-NNNN с пояснениями, или «нет».
   - `## Изученная документация` — список из шага 5.
   - `## Обсуждение` — Q/A из шага 6.
   - `## Решения` — итоги.
   - `## План` — из шага 7.
   - `## Журнал выполнения`, `## Доработки` — пусто.
   - `## История статусов` — `- 2026-04-18 created (ready)`.

5. **Обнови INDEX.md:** вызови утилиту «Перестроение секции «Задачи»» (см. SKILL.md).

### 9. Финал

```
Создана задача T-NNNN «<заголовок>», статус ready.
Эпики: E-01, E-02 (или «без эпика»)
Файл: .planning/tasks/T-NNNN-<slug>/TASK.md
Запусти /task do <NNNN> для выполнения.
```

## Запрещено

- Git-операции.
- Начинать реализацию кода — это делает `do`.
- Принимать решения за пользователя по открытым вопросам.
- Создавать дубликат задачи при ответе `a`.
