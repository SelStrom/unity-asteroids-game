# task — система управления задачами

Автономная система задач в `.planning/tasks/`. Не использует GSD, не трогает git, не использует фазы. Каждая задача — единый файл `TASK.md`, который накапливает всё состояние. Задачи группируются в эпики (необязательно).

## Общие правила (действуют во ВСЕХ режимах)

- **Язык всех файлов и сообщений пользователю:** русский.
- **Git:** НИКОГДА не вызывать `git add`, `git commit`, `git push`, `git stash`, `git reset`, `git checkout` и любые другие git-команды. Ни автоматически, ни по умолчанию. Пользователь сам решает, когда и что коммитить.
- **Расположение:**
  - Активные задачи: `.planning/tasks/T-NNNN-<slug>/TASK.md`
  - Архив: `.planning/tasks/archive/T-NNNN-<slug>/TASK.md`
  - Индекс: `.planning/tasks/INDEX.md`
- **Нумерация задач:** глобальная `T-0001`, `T-0002`, … Новый ID = `max(все T-NNNN в INDEX.md и в archive/) + 1`, 4 цифры с ведущими нулями.
- **Нумерация эпиков:** глобальная `E-01`, `E-02`, … 2 цифры с ведущим нулём. Новый `E-NN` = `max(все E-NN в INDEX.md) + 1`.
- **Slug:** 2–5 слов из заголовка в kebab-case, транслит русского в латиницу.
- **Текущая дата:** `2026-04-18` (из контекста CLAUDE.md). Используй её для `created:` и меток в логе.

## Короткие ID (работают во ВСЕХ режимах)

Пользователь может указывать задачу или эпик сокращённо. **Нормализуй ДО выполнения операции.**

**Задачи** (нужны в `do`, `review`, `epics`, `updocs`, `list <слово>`):
- `357` или `0357` → `T-0357`
- `T-5`, `t-5`, `T5`, `t5` → `T-0005`
- Правило: если строка матчится `^(?i)t?-?\d+$` → выдели цифры, pad до 4 знаков, префикс `T-`.

**Эпики** (нужны в `epic close/rename/reopen`, `epics set/add/remove`, `list E-NN`):
- `1` или `01` → `E-01`
- `E-1`, `e-1`, `E01`, `e1` → `E-01`
- Правило: если строка матчится `^(?i)e?-?\d+$` → цифры, pad до 2 знаков, префикс `E-`.

**Если аргумент — просто число без префикса,** определяй тип по контексту команды:
- `do`, `review`, `epics`, `updocs` → задача (`T-NNNN`).
- `epic close/rename/reopen` → эпик (`E-NN`).
- `list` → если перед числом стоит `E`/`e` или контекст явно про эпики — эпик, иначе трактуй как подстрочный поиск.

Если после нормализации ID не найден — сообщи: «Задача `T-NNNN` не найдена» (или «Эпик `E-NN` не найден») и остановись.

## Эпики (обзор)

- Хранятся ТОЛЬКО как таблица `## Эпики` в `INDEX.md`. Отдельных файлов эпиков нет.
- Задача связана с эпиками через frontmatter `epics: [E-01, E-02]`. Пусто → задача в секции «Без эпика».
- Одна задача может принадлежать нескольким эпикам → в секции `## Задачи` INDEX она дублируется в каждой соответствующей подсекции. Источник истины — frontmatter TASK.md.
- При ЛЮБОМ изменении (создание, статус, эпики, архивация) секция `## Задачи` INDEX.md перестраивается целиком по утилите ниже.

## Статусы задачи

`ready` → `executing` → `review` ⇄ `revise` → `done`

## Статусы эпика

`active` → `closed` (закрыть эпик можно только когда все его задачи `done`).

## Диспетчеризация по режимам

Первое слово аргументов команды определяет режим. Остальное — аргументы режима.

| Первое слово | Режим | Файл инструкций |
|--------------|-------|-----------------|
| `new` | создание задачи | `.claude/skills/task/modes/new.md` |
| `do` | выполнение | `.claude/skills/task/modes/do.md` |
| `review` | проверка/приёмка | `.claude/skills/task/modes/review.md` |
| `list` | список | `.claude/skills/task/modes/list.md` |
| `epic` | управление эпиками | `.claude/skills/task/modes/epic.md` |
| `epics` | эпики конкретной задачи | `.claude/skills/task/modes/epics.md` |
| `updocs` | обновление GSD-документов по итогам задачи | `.claude/skills/task/modes/updocs.md` |

### Логика запуска

1. **Нет аргументов** — спроси: «Что делаем — `new`, `do`, `review`, `list`, `epic`, `epics` или `updocs`?». Жди ответ.
2. **Первое слово распознано** — `Read` соответствующий `modes/*.md` и следуй ТОЛЬКО его инструкциям. Остальные слова — аргументы режима.
3. **Не распознано** — «Неизвестный режим `<слово>`. Доступны: new, do, review, list, epic, epics, updocs.» Жди.

## Утилита: перестроение секции «Задачи» в INDEX.md

Вызывается режимами `new`, `do`, `review`, `epics` при изменении структуры/статуса/эпиков задач. Также доступна напрямую через `/task epic list` для синхронизации счётчиков.

Алгоритм:

1. Прочитай таблицу `## Эпики` из INDEX.md → карта `{E-NN: {slug, title, status}}`.
2. Собери активные задачи: `Glob` `.planning/tasks/T-*/TASK.md`. Для каждой прочитай frontmatter: `id`, `status`, `slug`, `title`, `created`, `epics`, `related`.
3. Сгруппируй:
   - Для каждого E-NN — подсекция `### E-NN — <title эпика>` с таблицей задач где `E-NN ∈ epics`.
   - Подсекция `### Без эпика` — задачи с `epics: []`.
4. Задача с N эпиками ДУБЛИРУЕТСЯ в каждой соответствующей подсекции.
5. Перепиши секцию `## Задачи` целиком, сохранив всё выше и ниже этой секции нетронутым.
6. Формат строки таблицы:
   ```
   | T-NNNN | <статус> | <заголовок> | <создана> | <related или —> |
   ```

## Инициализация хранилища

Перед ЛЮБОЙ операцией проверь `.planning/tasks/INDEX.md`. Если нет:
1. `mkdir -p .planning/tasks/` (если папки нет).
2. Создай `INDEX.md` из `templates/INDEX.md.tmpl`.
