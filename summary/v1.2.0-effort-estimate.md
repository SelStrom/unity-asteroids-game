## Баги и ошибки Claude

За время разработки Claude допустил 5 ошибок, потребовавших исправления:

### 1. Забыл создать Rocket префаб (Phase 13)
- **Коммит:** `fix(13): create rocket prefab and assign to GameData.Rocket.Prefab`
- **Суть:** Код ссылался на `_configs.Rocket.Prefab`, но сам префаб не был создан в Unity. GameData.asset содержал пустую ссылку (None).
- **Причина:** План Phase 12 описывал CreateRocket factory, но не включал задачу на создание самого .prefab ассета. Claude написал код, который ожидал префаб, но не создал его.
- **Время на fix:** ~10 мин (создание через MCP)

### 2. Ошибки компиляции в Phase 14
- **Коммит:** `fix(14): исправить ошибки компиляции, настроить trail PS и GameData.asset через MCP`
- **Суть:** После добавления RocketData struct и trail кода проект не компилировался.
- **Причина:** Несогласованность между новыми полями в GameData.cs и существующим кодом в EntitiesCatalog. Claude написал код в двух файлах параллельно без проверки компиляции между задачами.
- **Время на fix:** ~8 мин

### 3. NullReferenceException в HudVisual (Phase 15)
- **Коммит:** `fix: null-guard для rocket SerializeField в HudVisual.OnConnected()`
- **Суть:** При запуске PlayMode — NullReferenceException на строке 40 HudVisual.cs. Новые SerializeField `_rocketAmmoCount` и `_rocketReloadTime` были null.
- **Причина:** Claude редактировал Main.unity напрямую (YAML), добавляя ссылки на TMP_Text объекты. Unity не подхватила изменения корректно — stripped PrefabInstance references не десериализовались. Решение: добавить null-guard + пересохранить сцену.
- **Время на fix:** ~15 мин (debug session + fix + UAT)

### 4. Trail ракеты невидим (Phase 14/15)
- **Коммит:** `fix: заменить Default-Particle на URP-материал для trail ракеты`
- **Суть:** Ракета летела без видимого следа. ParticleSystem работал, но частицы невидимы.
- **Причина:** Editor-скрипт RocketPrefabSetup создал ParticleSystem с материалом Default-Particle (Built-in Render Pipeline), который невидим в URP. Claude не учел, что проект использует URP и нужен URP-совместимый материал.
- **Время на fix:** ~18 мин (debug session + замена материала)

### 5. Score=0 вместо 50 в GameData.asset (Phase 14 -> Phase 16)
- **Коммит:** `fix(config): set Rocket Score=50 in GameData.asset (was 0)`
- **Суть:** RocketData.Score в ассете остался 0, хотя должен быть 50.
- **Причина:** Editor-скрипт Phase 14 должен был установить Score=50, но либо не выполнился, либо значение не сохранилось. Обнаружено только при аудите milestone.
- **Время на fix:** 30 сек (правка YAML)
